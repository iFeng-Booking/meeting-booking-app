# 智慧空间预约系统

一个连接 Supabase 数据库的可部署预约系统 demo，包含三个演示入口（顶部切换）：

- **电脑预约 / 后台管理**：员工预约 + 行政审批 + 空间管理 + 数据看板
- **手机端演示**：手机屏幕比例的预约/签到/我的预约
- **门口智慧屏演示**：模拟会议室门口的状态显示屏

## 本地运行（可选，非必须）
```
npm install
npm run dev
```

## 部署
本项目是纯前端（Vite + React），构建后是静态文件，可以直接部署到 Vercel / Netlify 等平台，
数据库使用 Supabase（`src/supabaseClient.js` 里已经配置好连接信息）。

## 数据库
需要在 Supabase 项目里执行以下两步：
1. 建表：见项目里的建表 SQL（spaces / bookings 两张表）
2. 灌入示例空间数据：运行 `seed.sql`

## ⚠️ 上线前必须做的事（当前仅适合内部演示/试点）
1. **打开 RLS（行级安全）并配置权限规则**：现在为了演示方便，两张表的 RLS 是关闭的，意味着任何拿到
   网址的人都能读写全部数据。正式给全公司使用前，必须重新开启 RLS，并配置"只有登录用户能读写"等规则。
2. **加登录鉴权**：目前没有登录系统，`CURRENT_USER` 是写死在代码里的示例用户（李婷 / 行政部），
   多人使用时每个人看到的"我的预约"其实是同一个人。上线前需要接入企业的登录方式（如企业微信/飞书/OA 账号）。
3. **通知渠道是模拟的**：邮件/短信/微信通知目前只是界面上的提示文字，没有真的发出去，需要接入
   真实的短信/邮件/企业IM服务商接口。
4. **中控/门禁/人脸识别硬件**：门口智慧屏目前是纯软件演示，真实落地需要对接楼宇的中控系统、
   门禁读卡器或人脸识别设备的 SDK。
